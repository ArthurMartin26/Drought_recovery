Terrain elevation data were sourced from the Shuttle Radar Topography Mission (SRTM) Global Land Digital Elevation Model (GL3, approximately 90 m spatial resolution) via the OpenTopography API. The data were downloaded as a GeoTIFF using a WGS84 (EPSG:4326) bounding box covering Nigeria. The raw raster was first validated by inspecting its spatial extent, resolution, coordinate reference system, and value ranges to ensure that it contains plausible elevation values across the study region. Nigeria administrative boundary vectors (ADM2) were then loaded and dissolved into a single national polygon, which was used to crop and mask the DEM so that only land area within Nigeria’s borders is retained. This step removes unnecessary buffer regions while preserving the full spatial detail of the terrain surface relevant to the analysis.
To enable meaningful computation of terrain derivatives, the Nigeria‑masked DEM was reprojected from geographic coordinates (latitude–longitude, EPSG:4326) into a projected coordinate reference system using Universal Transverse Mercator (UTM) Zone 32N (EPSG:32632). This reprojection is a critical methodological step. In geographic coordinate systems, distances are measured in degrees, which do not correspond to constant physical distances on the Earth’s surface; as a result, quantities such as slope, gradients, and distances cannot be interpreted correctly. In contrast, UTM is a locally conformal projected coordinate system in which horizontal units are metres, ensuring that spatial derivatives represent physically meaningful rates of change (e.g. metres of elevation change per metre of horizontal distance). Using a metric projection is therefore essential for deriving terrain‑based variables that enter the analysis as structural geographic covariates.
Slope was subsequently derived from the projected DEM using the terrain() function in the terra package with the argument v = "slope". Conceptually, slope is defined as the magnitude of the spatial gradient of elevation, i.e. the maximum rate of vertical change with respect to horizontal movement. Numerically, this is computed using finite‑difference approximations to the first partial derivatives of elevation in the east–west and north–south directions, which are then combined to form the gradient magnitude. The resulting quantity corresponds to the tangent of the local slope angle, and is reported here in degrees for ease of interpretation. This approach is standard in geographic information systems and ensures that slope is invariant to map orientation and captures overall terrain steepness rather than directional change along a single axis.
The resulting slope raster represents a time‑invariant measure of terrain steepness across Nigeria and is treated as a fixed structural characteristic of space. Together with mean elevation and other terrain‑derived measures, slope enters the analysis as part of the static spatial feature set (Zi), which captures long‑run physical and geographic conditions that may shape exposure and resilience to drought but are not themselves affected by short‑run climate shocks. These features will later be spatially aggregated to the analysis grid and used as inputs to the convolutional neural network, ensuring that learned spatial patterns reflect economically meaningful and physically grounded heterogeneity rather than artefacts of coordinate representation.
