# Updated README: Terrain Elevation and Slope (Zi)


Terrain elevation data were sourced from the Shuttle Radar Topography Mission (SRTM) Global Land Digital Elevation Model (GL3, approximately 90 m spatial resolution) via the OpenTopography API. The data were downloaded as a GeoTIFF using a WGS84 (EPSG:4326) bounding box covering Nigeria. The raw raster was first validated by inspecting its spatial extent, resolution, coordinate reference system, and value ranges to ensure that it contains plausible elevation values across the study region. Nigeria administrative boundary vectors (ADM2) were then loaded and dissolved into a single national polygon, which was used to crop and mask the DEM so that only land area within Nigeria’s borders is retained. This step removes unnecessary buffer regions while preserving the full spatial detail of the terrain surface relevant to the analysis.

To enable meaningful computation of terrain derivatives and ensure consistency with other spatial inputs in the project, the Nigeria‑masked DEM was reprojected from geographic coordinates (latitude–longitude, EPSG:4326) into an equal‑area projected coordinate reference system (Africa Albers, ESRI:102022). This projection is used throughout the project for all spatial variables entering the fixed geographic feature set (Zi). Unlike geographic coordinate systems, which express distances in degrees, ESRI:102022 uses metres as its horizontal units while preserving area exactly. This ensures that spatial derivatives are physically meaningful and that all Zi covariates are defined on a common metric grid, avoiding distortions arising from mismatched projections or cell areas.

Slope was subsequently derived from the reprojected DEM using the terrain() function in the terra package with the argument v = "slope". Conceptually, slope is defined as the magnitude of the spatial gradient of elevation, i.e. the maximum rate of vertical change with respect to horizontal movement. Numerically, this is computed using finite‑difference approximations to the first partial derivatives of elevation in the east–west and north–south directions, which are then combined to form the gradient magnitude. The resulting quantity is reported in degrees for ease of interpretation. This approach is standard in geographic information systems and ensures that slope captures overall terrain steepness rather than directional change along a single axis.

Both elevation and slope are subsequently aggregated to the common 10 km × 10 km equal‑area analysis grid used elsewhere in the project (for example, for rainfall, surface water, and market‑access measures). Treating terrain variables on the same grid ensures that all Zi inputs share identical spatial support and alignment, which is essential for coherent representation learning using the convolutional neural network.

The resulting elevation and slope rasters represent time‑invariant measures of Nigeria’s physical geography and are treated as fixed structural characteristics of space. Together with other geographic inputs, these variables enter the analysis as part of the latent spatial feature set (Zi), capturing long‑run physical conditions that shape exposure and resilience to drought but are not themselves affected by short‑run climate shocks. By harmonising projection and grid structure across all Zi channels, the analysis ensures that learned spatial patterns reflect meaningful geographic heterogeneity rather than artefacts of coordinate representation.
